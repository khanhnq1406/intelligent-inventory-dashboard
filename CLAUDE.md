## Project Overview

Intelligent Inventory Dashboard for dealership managers — real-time vehicle stock overview with aging stock identification (>90 days) and actionable insights. Monorepo with Go backend, Next.js frontend, PostgreSQL database.

## Architecture

**Three-tier monorepo** with shared types generated from a single OpenAPI spec:

```
api/openapi.yaml  →  oapi-codegen      →  backend/internal/handler/api.gen.go
                  →  openapi-typescript  →  frontend/src/lib/api/types.ts
```

- **Backend** (`backend/`): Go + Chi router + oapi-codegen. Layered: `handler/` → `service/` → `repository/` (pgx).
- **Frontend** (`frontend/`): Next.js 14 App Router + TanStack Query + shadcn/ui + Tailwind CSS.
- **Database**: PostgreSQL 16. Three tables: `dealerships`, `vehicles`, `vehicle_actions`.
- Aging stock is always computed (`NOW() - stocked_at > 90 days`), never stored.
- `vehicle_actions` is append-only (audit trail).

## Commands

```bash
# Full stack (Docker Compose)
docker compose up              # Start all services (PostgreSQL, backend, frontend)
docker compose down            # Stop all services

# Code generation (after editing api/openapi.yaml)
make generate                  # Generate both Go and TypeScript types
make generate-go               # Generate Go server code only
make generate-ts               # Generate TypeScript client types only

# Backend
cd backend
go run ./cmd/server            # Run backend server (port 8080)
go test ./...                  # Run all backend tests
go test ./internal/service/... # Run service layer tests only
go test -v -run TestName ./internal/service/...  # Run a single test

# Frontend
cd frontend
npm install                    # Install dependencies
npm run dev                    # Run dev server (port 3000)
npm test                       # Run frontend tests
npm run lint                   # Lint frontend code

# Database migrations
make migrate-up                # Apply all pending migrations
make migrate-down              # Rollback last migration
```

## API Convention

All endpoints are prefixed with `/api/v1`. The OpenAPI spec (`api/openapi.yaml`) is the single source of truth — edit it first, then run `make generate`. Never manually edit generated files (`api.gen.go`, `types.ts`).

## Backend Patterns

- **Handler**: Generated by oapi-codegen from OpenAPI. Custom logic goes in the implementation that satisfies the generated interface.
- **Service**: Business logic layer. Aging calculation, validation, and orchestration live here.
- **Repository**: Database access only. Uses pgx with parameterized queries. No business logic.
- **Middleware**: Request ID, structured logging (slog), CORS, request duration tracking.
- **Logging**: Use `slog` (stdlib). JSON format. Always include `request_id` in context.

## Frontend Patterns

- **Data fetching**: All API calls go through TanStack Query hooks in `hooks/`. Use generated TypeScript types from `lib/api/`.
- **Components**: shadcn/ui primitives in `components/ui/`, composed into feature components in `components/`.
- **Pages**: Next.js App Router in `app/`. Four routes: `/` (dashboard), `/inventory`, `/aging`, `/vehicles/[id]`.
